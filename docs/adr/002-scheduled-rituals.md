# ADR-002: טקסים מתוזמנים (Devlog שבועי + אבן-דרך חודשית)

- **תאריך:** 2026-04-19
- **סטטוס:** מאושר
- **אחראית:** Marina
- **מחליף:** —
- **קשור:** [ADR-001](001-layered-backlog-system.md) §סיכונים (ציות לטקסים)

## הקשר

`CLAUDE.md §קצב טקסים` דורש ארטיפקטים כתובים עמידים:

| קצב | ארטיפקט | מיקום |
|---|---|---|
| שבועי | Weeknote | `docs/devlog/YYYY-W##.md` |
| חודשי | סקירת אבן-דרך | `docs/milestones/YYYY-MM.md` |
| לכל release | רשומת CHANGELOG | `CHANGELOG.md` |
| לכל החלטה | ADR | `docs/adr/NNN-title.md` |
| לכל תקלה | Postmortem | `docs/postmortems/YYYY-MM-DD-title.md` |

ADR-001 זיהה **ציות לטקסים** כסיכון הפתוח העליון של מערכת ה-backlog
בת 4 השכבות: "שכבות 3–4 דורשות מ-Marina להריץ סקריפטים
שבועית. אם היא לא, הם נרקבים."

אותו דפוס כשל חל על weeknotes וסקירות אבן-דרך. מתחזקת סולו
שמסתמכת על זיכרון כדי לעמוד בקצב שבועי תסחף
תוך 2–3 שבועות. הוכחה: שבוע 1 (השבוע הזה) כמעט פספס
weeknote; רק §צ'ק-ליסט פתיחת שיחה ב-CLAUDE.md תפס זאת.

שני פערי טקס לא-פתורים:

1. **Weeknotes** — לא קיים טריגר להתחיל טיוטה כל יום שישי.
2. **אבני-דרך חודשיות** — לא קיים טריגר להתחיל טיוטה ב-1 לחודש.

## החלטה

הוספת שני סוכנים מרוחקים מתוזמנים דרך skill ה-`/schedule` של Claude Code.
כל סוכן מתעורר ב-cron שלו, קורא את המקורות הרלוונטיים, ומייצר
**טיוטה** (לא ארטיפקט סופי) במיקום הנכון.

### Schedule 1 — Devlog שבועי

- **Cron:** שישי 18:00 Asia/Jerusalem.
- **הגדרת שבוע:** ראשון-שבת (7 ימים). הסוכן רץ שישי 18:00 כי זה
  סוף יום העבודה האחרון בשבוע הישראלי; שבת בלי פעילות קוד
  אז אין דאטה חסר (הובהר 2026-04-23, Marina).
- **מטרה:** טיוטה של `docs/devlog/YYYY-W##.md` לשבוע הנוכחי (ראשון-שבת).
- **כניסות:** `git log` לשבוע; diff ב-`tasks/BACKLOG.md` וב-
  `tasks/FEEDBACK-LOG.md`; תיקיות משימה חדשות או סגורות; ADRs חדשים.
- **סעיפי תבנית:** עלה · הפתעות · תקוע · הבא · Snapshot מטריקות.
- **חוזה output:** קובץ committed ל-branch `devlog/YYYY-W##`
  עם PR פתוח ל-Marina לערוך ולמזג. **טיוטות לא-ערוכות לא
  עולות** — לפי CLAUDE.md §Flow טיוטת Weeknote.

### Schedule 2 — אבן-דרך חודשית

- **Cron:** ה-1 לחודש, 09:00 Asia/Jerusalem.
- **מטרה:** טיוטה של `docs/milestones/YYYY-MM.md` לחודש הקודם.
- **כניסות:** כל ה-weeknotes בחודש; diff של CHANGELOG לחודש;
  ADRs שנכתבו בחודש; פריטי BACKLOG שנפתחו לעומת נסגרו.
- **סעיפי תבנית:** עלה · ראיות (מטריקות) · החלטות (ADRs)
  · העברה · לקחי רטרוספקטיבה.
- **חוזה output:** זהה לשבועי — PR עם טיוטה; Marina עורכת.

### לא מתוזמנים

- **ADRs** — נכתבים ברגע ההחלטה, לא בקצב.
- **רשומות CHANGELOG** — נכתבות ב-commit העלאה.
- **Postmortems** — נכתבים אחרי תקלה, תוך 72 שעות.

אוטומציית קצב שמורה לארטיפקטים ש-(א) חוזרים ו-(ב)
סובלים מסחף תחת אכיפה של זיכרון בלבד.

## אלטרנטיבות שנשקלו

- **Hooks של Claude Code על אירועי `Stop`/`SessionStart`.** יכולים להזכיר
  ל-Marina ביום שישי. נדחה: דורש ש-Marina תפתח את Claude Code ביום
  שישי — בדיוק כשל הזיכרון שאנחנו מנסים להסיר. סוכן מתוזמן
  מרוחק רץ ללא תלות בכך.
- **תזכורות יומן (Google Calendar).** נדחה: מזכיר ל-Marina
  לכתוב את ה-weeknote; לא מייצר טיוטה. 80% מהערך הוא
  הטיוטה, לא התזכורת.
- **Cron של GitHub Actions ב-push.** נדחה: רץ רק כשה-repo
  רואה push. שבועות שקטים = cron שקט = בלי weeknote — המקרה הגרוע.
- **לדלג על אוטומציה; להסתמך על צ'ק-ליסט פתיחת שיחה.** המצב הנוכחי.
  תפס את הפער של שבוע 17 אבל רק כי Marina פתחה את Claude Code.
  לא מתכלל מעבר ל-2–3 שבועות של חוסר-פעילות.
- **טיוטת weeknote בסוף סשן במקום ביום שישי.** נדחה:
  קושר את קצב ה-weeknote לקצב העבודה. שבוע עם סשן אחד יקבל
  weeknote אחד; שבוע עם חמישה יקבל חמישה. הקצב חייב להיות זמני,
  לא מופעל-פעילות.

## השלכות

### נפתח

- שני ארטיפקטים שחיו קודם בזיכרון יש להם עכשיו טריגר מאוכף מכונה.
- הטקס השבועי של Marina בן 5 דקות = לערוך את הטיוטה, לא לכתוב מאפס.
- סקירות אבן-דרך חודשיות הופכות ריאליות לפרויקט סולו. בלי
  אוטומציה, היו מדולגות.
- הדפוס מתכלל: כל קצב עתידי (רטרו רבעוני, סקירה
  שנתית) במרחק רשומת `/schedule` שלישית.

### נסגר

- הסיכון העליון של ADR-001 ("ציות לטקסים. שכבות 3–4 דורשות מ-Marina
  להריץ סקריפטים שבועית").
- "שכחתי שזה יום שישי" כתירוץ מקובל.

### סיכונים / שאלות פתוחות

- **סחף איכות סוכן.** אם איכות הטיוטה גרועה, Marina עורכת
  הרבה → הטקס מרגיש כעבודה נוספת → היא מפסיקה למזג את ה-PR →
  PRs מצטברים → האוטומציה מתה מלחץ backlog. מיטיגציה:
  סקירה רבעונית של איכות טיוטות; כוונון הפרומפט; קבלה
  שטיוטות שבועות 1–4 יהיו גרועות יותר משבוע 20.
- **נקודות עיוורות של cron.** מכשיר כבוי, לפטופ ישן, בלי רשת ביום שישי
  18:00 → הרצה שפוספסה. התנהגות skill ה-`/schedule` בהרצות שפוספסו חייבת
  להיבדק; אם אינו ממלא רטרואקטיבית, להוסיף retry ביום ראשון 09:00.
- **עלות.** שני סוכנים מתוזמנים × 4 שבועי + 1 חודשי = 5 הרצות סוכן
  לחודש. מקובל בשימוש הנוכחי; לבחון שוב אם גדל
  ליצירת תוכן העשרה/סביבה.
- **סחף אזור זמן.** DST של ירושלים משתנה פעמיים בשנה. `/schedule`
  חייב לכבד `Asia/Jerusalem` ולא UTC. לאמת בגבול DST ראשון.
